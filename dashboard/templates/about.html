About The Data

Dataset

Le dashboard est construit sur le dataset public Online Retail II, mis à disposition par l'UCI Machine Learning Repository (archive.ics.uci.edu). Il contient les transactions d'un e-commerce britannique spécialisé dans les cadeaux et articles occasion, sur la période 1ᵉʳ décembre 2009 → 9 décembre 2011.

Pipeline de traitement

Pattern lakehouse en 3 schémas (raw → clean → analytics) orchestré par un DAG Apache Airflow en 4 tasks séquentielles :

  1. ingest_xlsx — XLSX → CSV temporaire → COPY FROM STDIN dans raw.online_retail
  2. transform_clean_rfm — cascade exclusive de cleaning (4 buckets) puis calcul RFM via NTILE(5)
  3. load_segments — assignation des 11 segments via UPDATE CASE
  4. compute_rfm_history — extension snapshot mensuel (Sprint 5)

Stack : Postgres 15, Python 3.11, Apache Airflow 2.9, Flask + reveal.js, Plotly, le tout conteneurisé via Docker Compose.

Cleaning : cascade exclusive en 4 buckets

La cascade route les lignes métier vers un seul bucket grâce à un ordre strictement décroissant en spécificité. Elle n'est toutefois pas exhaustive : les transactions anonymes et autres anomalies hors scope RFM v1 restent écartées.

  • clean.stock_movements — mouvements internes (description NULL + customer_id NULL + price = 0)
  • clean.cancellations — factures préfixées C*
  • clean.non_product_lines — codes produits techniques (POST, DOT, BANK CHARGES, ADJUST, etc.)
  • clean.sales — ventes exploitables pour le RFM (customer_id renseigné, quantité/prix > 0)

Volumétrie routée : ~4 382 / ~19 494 / ~4 805 / ~802 637 lignes. Environ 236 k lignes restent hors clean.*, surtout des transactions anonymes sans customer_id.

Inspiration

Le dashboard v2 s'inspire de la structure du dashboard RFM JustDataPlease (un Looker Studio embarqué construit sur les données d'une chaîne de restaurants anonyme). 16 graphiques sur 18 ont été reproduits sur Online Retail II avec quelques adaptations sur les bins monétaires (calibrés sur nos percentiles 20/40/60/80 plutôt que sur les seuils $ d'origine). Le détail de l'audit est dans docs/dashboard_v2_audit.md.
